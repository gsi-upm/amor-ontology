# AMOR competency-question evaluation

This evaluation operationalises ten competency questions against the compact
AMOR example. Each query is standalone: every `.rq` file declares all prefixes
that it uses.

## Evaluation graph

The graph is exactly the RDF union listed in `graph-manifest.txt`:

```text
src/amor/amor.ttl
src/bhv/bhv.ttl
src/mft/mft.ttl
src/amor/models/bhv/amor-bhv.ttl
src/amor/models/mft/amor-mft.ttl
src/amor/examples/amor-examples.ttl
```

The experimental ontology is intentionally excluded. Execute every query, or
selected query identifiers, from the repository root:

```bash
uv run --frozen python evaluation/run_queries.py
uv run --frozen python evaluation/run_queries.py --summary
uv run --frozen python evaluation/run_queries.py CQ-C1 CQ-M1
```

The automated validation uses the same manifest and runs offline:

```bash
UV_CACHE_DIR=/tmp/uv-cache uv run --offline --frozen pytest -q
```

## Traceability

| ID | Competency question | Principal classes and properties | Query | Stable evidence | Test |
|---|---|---|---|---|---|
| CQ-C1 | What resource was analysed in a moral-value analysis? | `amor:MoralValueAnalysis`, `amor:analysed` | `competency-questions/CQ-C1.rq` | The four moral analyses bind to `amor-examples:news1`. | `test_cq_c1_analysed_resource` |
| CQ-C2 | Which moral-value annotations were generated by an analysis? | `amor:MoralValueAnalysis`, `prov:generated`, `amor:MoralValueAnnotation` | `competency-questions/CQ-C2.rq` | Each of the four analyses binds to its specific moral annotation. | `test_cq_c2_generated_annotations` |
| CQ-C3 | What resource is targeted by a moral-value annotation? | `amor:MoralValueAnnotation`, `oa:hasTarget` | `competency-questions/CQ-C3.rq` | All four moral annotations target `amor-examples:news1`. | `test_cq_c3_annotation_targets` |
| CQ-C4 | Which moral-value model was used in an analysis? | `amor:MoralValueAnalysis`, `amor:usedMoralValueModel` | `competency-questions/CQ-C4.rq` | `analysis2` uses BHV level 3; the other analyses use the consolidated MFT model. | `test_cq_c4_moral_models` |
| CQ-C5 | Which moral-value category was assigned by an annotation? | `amor:MoralValueAnnotation`, `amor:hasMoralValueCategory` | `competency-questions/CQ-C5.rq` | The bindings are Liberty, Conservation, Care, and Authority on their respective annotations. | `test_cq_c5_assigned_categories` |
| CQ-C6 | Which categories are included in a moral-value model? | `amor:MoralValueModel`, `amor:hasMoralValueCategory` | `competency-questions/CQ-C6.rq` | The five BHV model individuals and consolidated MFT model are returned; Conservation belongs to BHV level 3 and Fairness belongs to MFT. | `test_cq_c6_model_memberships` |
| CQ-C7 | Which agent was responsible for a moral-value analysis? | `amor:MoralValueAnalysis`, `prov:wasAssociatedWith` | `competency-questions/CQ-C7.rq` | The four analyses bind to `human1`, `human2`, `robot1`, and `robot2`, respectively. | `test_cq_c7_responsible_agents` |
| CQ-C8 | What confidence score is associated with a moral-value annotation? | `amor:MoralValueAnnotation`, `itsrdf:taConfidence` | `competency-questions/CQ-C8.rq` | Each moral annotation binds to its asserted confidence literal. | `test_cq_c8_confidence_scores` |
| CQ-C9 | Which machine-learning model was used in a moral-value analysis? | `amor:MoralValueAnalysis`, `amor:usedMLModel`, `mls:Model` | `competency-questions/CQ-C9.rq` | `analysis3` uses `model1` and `moralAnalysis21` uses `model4`. | `test_cq_c9_machine_learning_models` |
| CQ-M1 | What MFT-specific polarity and intensity are associated with an annotation? | `amor:MoralValueAnnotation`, `amor-mft:hasPolarity`, `amor-mft:hasPolarityIntensity` | `competency-questions/CQ-M1.rq` | Three annotations bind to polarity; `annotation3` and `moralAnnotation23` also bind to their asserted intensities, while `annotation1` has no asserted intensity. | `test_cq_m1_polarity_and_intensity` |

Except for CQ-C6, the tests compare the complete expected binding sets (4 rows
for C1-C5 and C7-C8, 2 for C9, and 3 for M1). CQ-C6 deliberately avoids a
brittle aggregate count and instead checks the stable model individuals and
specific category memberships described above.
