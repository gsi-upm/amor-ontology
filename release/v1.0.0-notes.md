# AMOR ontology v1.0.0

AMOR v1.0.0 provides the reproducible ontology release accompanying the AMOR
paper.

## Included

- Normative AMOR, BHV, MFT, AMOR-BHV, AMOR-MFT, and compact example Turtle sources.
- Ten standalone competency-question SPARQL queries with traceability and expected bindings.
- Offline RDF syntax, model, example, competency-question, and generated-documentation validation.
- Reproducible WIDOCO 1.4.25 generation pinned by container digest.
- SHA-256 hashes for the normative paper-facing artefacts.

## Verification

- Release reference: annotated tag `v1.0.0`, resolving to the final release commit.
- Offline validation: `86 passed` with Python 3.12.7 and the locked uv environment.
- Competency questions: all ten executed; stable expected bindings and model memberships passed.
- CI: GitHub Actions validation completed successfully before publication.
- Documentation: generated from the tagged commit with WIDOCO 1.4.25 image digest
  `sha256:ef0548a0522d26cc73f3c3ecdb265cbed59aab9eeaa51f18a6096acbc239a41c`.
- Publication: the six canonical endpoints under
  `https://gsi.upm.es/ontologies/` returned parseable Turtle and HTML; the
